# RFQフロー仕様（MVP）

バージョン: 0.3
作成者: Yasuhiro Ueda
更新日: 2026-03-09

---

## 0. 用語定義（Roles）
| ロール | 役割 |
|--------|------|
| Requester（起案者） | 購買依頼の起案、仕様書アップロード、品質/開発等の評価入力 |
| Buyer | RFQ/RFP全体取りまとめ、配信実行、見積回収トラッキング、比較表・集計、最終推薦案作成 |
| Approver | 金額レンジに応じた承認（課長/部長/役員） |
| Procurement Finance | 見積・効果（Savings/TCO等）整合チェックと承認 |
| Supplier | 見積提出（標準フォーマット必須） |

---

## 1. 見積社数ルール
| 予算（購入単価） | 必要見積社数 |
|----------------|------------|
| 10万円以下 | 2社見積で可 |
| 10万円超〜100万円未満 | 3社推奨（ポリシー確定待ち） |
| 100万円以上 | 3社見積必須 |

---

## 2. 起案（Requester）

### 2.1 入力項目
- 案件名、背景、目的、納期希望、希望納入先（事業所/拠点）
- 予算（購入単価）
- コモディティ分類：中分類・小分類
- 候補サプライヤー選択（予算に応じて2社または3社）
- 仕様書（PDF推奨）：画面からアップロード（必須）

### 2.2 自動添付（System）
- **NDA**：全案件に自動添付（必須）
- **見積明細フォーマット（Excel）**：コモディティ中分類・小分類の選択に応じて自動生成

---

## 3. バイヤーレビュー & RFP発行（Buyer）

### 3.1 バイヤー確認（配信前チェック）
- 候補サプライヤー（2社 or 3社）
- 予算と見積社数ルール適合
- 必要書類の揃い：仕様書・RFP本文・見積明細フォーマット・NDA

### 3.2 配信実行
- 「展開ボタン」を押すとRFPが候補サプライヤーへ自動送信
- 送信内容：RFP本文・仕様書・見積明細フォーマット（Excel）・NDA

---

## 4. 見積回収トラッキング

- 見積提出期限：**送信日から14日**
- 受領状況管理：未受領 / 受領済 / 要修正
- 期限超過時：再促メール自動送信

---

## 5. 見積フォーマットチェック

- 標準フォーマット（見積明細Excel）の提出が必須
- 未提出・不備時：再提出要求メール自動送信

---

## 6. 見積明細の読み込み・集約・競争力評価

### 6.1 自動処理
- 標準見積明細（Excel）を自動取り込み
- 案件単位で集約・サプライヤー間比較表を生成

### 6.2 QCJDM評価（5段階: 1〜5点、5が良い）
| 評価軸 | 担当 | 方法 |
|--------|------|------|
| Cost（コスト） | System | 見積数値から自動スコアリング |
| Quality（品質） | Requester | 手動入力 |
| Job（対応力） | Requester | 手動入力 |
| Development（開発力） | Requester | 手動入力 |
| Management（管理力） | Requester | 手動入力 |

### 6.3 ゲーティング条件
- **いずれかの評価が「1」の場合、当該サプライヤーを提案不可に自動除外**

---

## 7. 承認提案（One Pager）& 承認フロー

### 7.1 One Pager自動生成（必須項目）
- 案件概要：案件名、目的、背景、納入先、希望納期、予算
- 候補サプライヤー一覧と受領状況
- 見積数値サマリ：合計金額、主要単価、価格比較
- QCJDM評価結果（サプライヤー別）
- 推奨サプライヤー（1社）と推奨理由（3点以内）
- リスク/留意事項
- 添付一覧

### 7.2 承認ルート（自動ルーティング）
1. Requester（起案内容最終確認）
2. Requester上長承認（金額レンジで自動判定）
3. Procurement Finance 承認
4. 必要に応じて上位承認（部長/役員）

### 7.3 金額レンジ別決裁権限
| 予算 | 決裁者 |
|------|--------|
| 〜10万円 | 課長 |
| 10万円〜100万円 | 部長 |
| 100万円超 | 役員 |

**承認者特定方法（自動）**
- `manager_employee_id` で上長を特定
- `job_level` / `job_title` で課長/部長/役員を判定
- Procurement Finance：部門=Finance、ロール=Approverの担当者

---

## 8. 画面・機能要件（最低限）

| 画面 | 機能 |
|------|------|
| 起案画面 | 分類（中/小）・予算・候補サプライヤー選択（2/3社強制）・仕様書アップロード |
| バイヤー画面 | 配信前チェック・展開ボタン・送信履歴 |
| 回収トラッキング | 受領状況・期限・未受領自動再促 |
| 見積集計 | 明細取り込み・比較表・コスト自動スコア |
| QCJDM入力 | Requester入力欄（5段階）・ゲーティング自動除外 |
| One Pager生成 | 評価結果＋見積数値＋推奨理由を1ページ自動生成、承認回付 |
| 承認ルート自動設定 | 金額レンジで課長/部長/役員を自動選定 |
